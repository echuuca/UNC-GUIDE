Echuuca Exploit Documentation

ENVIRONMENT

getgenv

<table> getgenv(<none>)

Returns the global environment table for Echuuca, which is shared across all scripts in the same exploit session. This allows for the storage and access of functions and variables globally, facilitating communication between scripts.

getrenv

<table> getrenv(<none>)

Returns the table of all Roblox global variables, providing access to the global scope. This function allows interaction with Roblox's built-in functions and properties, enabling script-level access to global data.

getsenv

<table> getsenv(Object<LocalScript> script)

Returns the global environment of the specified LocalScript, allowing access to its environment. This function is useful for modifying the script's global state and managing script-specific variables effectively.

getmenv

<table> getmenv(Object<ModuleScript> script)

Returns the global environment of the specified ModuleScript, similar to require. This function allows for the management of module-specific global state, enabling better organization of module data.

gettenv

<table> gettenv(thread thread)

Returns the environment table of the specified thread, allowing access to its state. This function is useful for managing thread-specific variables and ensuring proper isolation of thread data.

getfenv

<table> getfenv(<none>)

Returns the environment of the current function or the specified function. This is useful for accessing local variables and functions defined in that environment, facilitating scope management.

setfenv

<Variant> setfenv(Variant object, table env)

Sets the environment of the specified function or object to the provided table. This allows for changing the scope in which the function operates, enabling custom variable access and management.

getscriptenvs

<table> getscriptenvs(<none>)

Returns a table containing all scripts and their corresponding global environments. The keys are the scripts, and the values are their respective environments, enabling easy access to multiple script states and facilitating script management.
